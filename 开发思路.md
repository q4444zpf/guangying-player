### 2026-03-09：初始设计与实现概览

- **目标功能**：
  - 在 Windows 上开发一个全屏播放器，支持多种视频格式，以及“图片 + 音频”组合播放。
  - 通过 WebAPI 控制播放（播放/暂停/停止/重新播放）以及切换播放内容（视频或图片+音频）。
  - 通过 WebAPI 对电脑执行待机（睡眠）、关机、重启，以及关屏/亮屏等操作（真正的“远程唤醒”需另用 Wake-on-LAN 等方案）。

- **主要技术栈**：
  - 语言与运行时：C# + .NET 8（桌面目标 `net8.0-windows`）。
  - UI：WPF，全屏无边框窗口。
  - 媒体播放：LibVLC + LibVLCSharp.WPF（`VideoView` 控件输出视频画面）。
  - WebAPI：在同一进程内自托管 ASP.NET Core Minimal API（Kestrel）。

- **项目结构**（当前）：
  - 解决方案：`MediaControlPlayer.sln`
  - WPF 项目：`MediaControlPlayer.App`
    - `App.xaml` / `App.xaml.cs`：应用入口，负责加载配置、创建主窗口和服务，并启动/停止 WebAPI。
    - `MainWindow.xaml` / `MainWindow.xaml.cs`：全屏播放器窗口，包含：
      - `VideoViewControl`：LibVLCSharp.WPF 视频输出控件。
      - `ImageOverlay`：图片显示层，用于图片+音频模式。
    - `Services/PlayerService.cs`：封装 LibVLC 的播放逻辑，提供：
      - `PlayVideo(path)`：播放视频文件。
      - `PlayImageWithAudio(imagePath, audioPath)`：切换到图片模式并播放音频。
      - `Pause()` / `Resume()` / `Stop()` / `Restart()`。
      - 内部通过 `Dispatcher` 调用 `MainWindow.ShowVideoMode` / `ShowImageMode`，确保 UI 线程安全。
    - `Services/PowerService.cs`：封装电源操作：
      - 使用 `shutdown` 命令实现关机 `/s` 与重启 `/r`。
      - 通过 `SetSuspendState` 进行睡眠（待机）。
      - 使用 `SendMessage` + `SC_MONITORPOWER` 实现关屏/亮屏。
    - `Services/WebApiHost.cs`：在 WPF 进程中启动 Minimal API：
      - 播放控制接口：
        - `POST /api/player/play`（请求体为 `PlayRequest`，`Type=Video` 或 `ImageWithAudio`）。
        - `POST /api/player/pause|resume|restart|stop`。
      - 电源控制接口：
        - `POST /api/system/shutdown|reboot|sleep|display-off|display-on`。
      - 运行状态：
        - `GET /api/health`。
    - `Models/MediaType.cs` / `Models/PlayRequest.cs`：WebAPI 请求模型与枚举。
    - `Config/AppSettings.json`：当前用于配置 WebAPI 监听地址（默认 `http://0.0.0.0:5000`）和媒体根目录（预留扩展）。

- **当前状态与验证**：
  - 解决方案可以成功 `dotnet build`。
  - 已 `dotnet publish` 生成 `publish` 目录，可用于部署。
  - 应用启动后会：
    - 创建全屏 WPF 播放窗口。
    - 初始化 LibVLC 播放器。
    - 读取 `Config/AppSettings.json` 并在指定 URL 启动 WebAPI。
  - 外部可通过 HTTP 请求控制播放与电源（注意：真正睡眠或关机后，进程停止，无法再通过 HTTP 自行唤醒，需要额外的 Wake-on-LAN/硬件方案）。

### 2026-03-09：代码审查与 Bug 修复记录

- **修复 MainWindow 与 PlayerService 重复实例化问题**：
  - 原来 `MainWindow` 构造函数内部会 `new PlayerService(...)` 并调用 `AttachMainWindow`，而 `App.OnStartup` 中也会新建一个 `PlayerService` 并再次 `AttachMainWindow`，导致同时存在两个播放器实例，容易引发行为不一致。
  - 调整后：
    - `MainWindow` 不再持有或创建 `PlayerService`，只负责展示 UI 和提供 `ShowVideoMode` / `ShowImageMode`。
    - `App.OnStartup` 中创建唯一的 `PlayerService` 实例，并在创建 `MainWindow` 之后调用 `AttachMainWindow` 进行绑定。

- **修复 StartupUri 导致的额外窗口与生命周期混乱问题**：
  - 原先 `App.xaml` 中设置了 `StartupUri="MainWindow.xaml"`，同时又在 `OnStartup` 里手动创建并显示 `MainWindow`，会导致 WPF 启动时自动创建一个隐式窗口，再额外创建一个显式窗口。
  - 现在移除了 `StartupUri`，只保留 `OnStartup` 中的手动创建逻辑，应用窗口生命周期完全由代码控制。

- **简化 PlayerService 构造与类型定义**：
  - 删除了 `PlayerService` 内部未使用的 `MediaType` 枚举，统一使用 `Models.MediaType` 表达媒体类型，避免混淆。
  - `PlayerService` 构造函数不再依赖 `Application` 实例（内部未实际使用），改为无参构造，减少耦合。

- **确保配置文件在发布后可用**：
  - 应用在运行时通过 `AppDomain.CurrentDomain.BaseDirectory/Config/AppSettings.json` 读取配置，但默认情况下该 JSON 不会随构建/发布被复制到输出目录。
  - 在项目文件 `MediaControlPlayer.App.csproj` 中为 `Config\AppSettings.json` 增加 `CopyToOutputDirectory=Always` 配置，保证调试和发布环境都能加载到配置。

### 2026-03-09：VideoView 鼠标事件与标题栏呼出修复

- **问题现象**：
  - 视频播放后，无法在应用端停止播放。
  - 鼠标移入顶端无法呼出标题栏。
  - 点击视频区域无法切换播放/暂停。

- **原因分析**：
  - LibVLCSharp.WPF 的 `VideoView` 存在 **airspace 问题**：使用 DirectX 渲染时，会拦截鼠标事件，导致 WPF 无法收到 `MouseMove`、`MouseLeftButtonUp` 等事件。
  - 原先依赖 `Window_MouseMove` 和 `VideoViewControl_MouseLeftButtonUp`，在视频播放时均无法触发。

- **解决方案**：
  - 在 `VideoView` 上方添加透明覆盖层 `VideoOverlay`（`Background="#01000000"`），置于 `Panel.ZIndex="1"`，覆盖 DirectX 渲染区域，由 WPF 控件接收鼠标事件。
  - 在 `VideoOverlay` 上绑定 `MouseMove`、`MouseLeftButtonUp`：当鼠标 Y ≤ 8px 时显示标题栏，点击时调用 `TogglePlayPause`。
  - 为 `ImageOverlay` 增加 `MouseMove`，图片模式下同样支持顶部呼出标题栏。
  - 在标题栏增加「■ 停止」按钮，呼出标题栏后可点击停止播放。

- **涉及文件**：
  - `MainWindow.xaml`：新增 `VideoOverlay`，为 `ImageOverlay` 增加 `MouseMove`，新增 `StopButton`。
  - `MainWindow.xaml.cs`：实现 `VideoOverlay_MouseMove`、`VideoOverlay_MouseLeftButtonUp`、`ImageOverlay_MouseMove`、`StopButton_Click`。

### 2026-03-09：VideoView 覆盖层 airspace 修复（覆盖层移入内部）

- **问题现象**：视频播放时，`VideoOverlay` 与 `PlayButtonOverlay` 无法置于视频之上，点击无效，无法暂停。

- **原因分析**：
  - 非线程问题。LibVLCSharp.WPF 的 `VideoView` 底层为 Win32 原生窗口，存在 **airspace 限制**：同级 WPF 控件无法叠在原生窗口之上，`Panel.ZIndex` 无效。
  - 原生视频窗口始终位于同级 WPF 元素之上，导致覆盖层收不到鼠标事件。

- **解决方案**：
  - 将 `VideoOverlay` 和 `PlayButtonOverlay` 从 `VideoView` 的**同级**改为 **VideoView 的子元素**。
  - LibVLCSharp.WPF 通过 ForegroundWindow 机制，将 VideoView 的子内容渲染在视频之上，从而正确接收鼠标事件。

- **涉及文件**：
  - `MainWindow.xaml`：将 `VideoOverlay`、`PlayButtonOverlay` 移入 `VideoViewControl` 内部，作为其子 Grid 的子元素。

### 2026-03-10：播放按钮图标显示时机修复

- **问题现象**：播放中显示播放图标，暂停中不显示图标（与预期相反）。

- **原因分析**：LibVLC 的 `IsPlaying` 在调用 `Pause()`/`Play()` 后存在异步更新，`UpdatePlayButtonVisibility()` 立即执行时读到的是旧状态。

- **解决方案**：新增 `DeferredUpdatePlayButtonVisibility()`，在 `UpdatePlayButtonVisibility` 前延迟 80ms 再执行，确保 LibVLC 已更新状态。`ShowVideoMode`、`ShowImageMode`、`TogglePlayPauseFromUi` 均改为调用 `DeferredUpdatePlayButtonVisibility()`。

### 2026-03-10：底部播放进度条

- **需求**：播放中在底部显示播放进度条，可拖动滑块调整进度，并显示当前时间与总时长。

  - **2026-03-10 更新**：进度条改为鼠标移入播放窗口后再显示；移出窗口时隐藏。通过 `Window_MouseEnter`、`Window_MouseLeave`、`VideoOverlay_MouseMove`、`ImageOverlay_MouseMove`、`Window_MouseMove` 控制可见性。因 VideoView 的 airspace 导致 `Window.MouseLeave` 不触发、`Mouse.GetPosition` 在原生区域上不可靠，改用 **Win32 GetCursorPos** 获取屏幕坐标，结合 `PointToScreen` 的窗口边界判断鼠标是否在窗口内；`ProgressTimer_Tick`、`VideoOverlay_MouseLeave`、`ImageOverlay_MouseLeave` 均调用 `IsMouseInsideWindow()` 检测。

- **涉及文件**：
  - `MainWindow.xaml`：新增 `ProgressBarRoot`、`ProgressSlider`、`CurrentTimeText`、`TotalTimeText`。
  - `MainWindow.xaml.cs`：`ProgressTimer_Tick`、`ProgressSlider_ValueChanged`、`PreviewMouseLeftButtonDown/Up`、`FormatTime`。
  - `PlayerService.cs`：新增 `Time`、`Length`、`Position`、`Seek`、`SeekToTime`。
  - **2026-03-10 更新**：在进度条后添加音量滑块（0–100）与静音按钮；`PlayerService` 新增 `Volume`、`IsMuted`、`ToggleMute`。
  - **2026-03-10 更新**：LibVLC 暂停时 `setPosition` 不更新画面：`Seek`、`SeekToTime` 改为 `SeekWhenPaused`：先 `Play()`，80ms 后执行 seek 与 `Pause()`，避免同步执行导致后续无法恢复播放；期间静音。若仍无法恢复播放，可尝试延长延迟或改用其他方案。
